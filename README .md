Answers to Questions

1. What is the difference between getElementById, getElementsByClassName, and querySelector / querySelectorAll?

getElementByID : এটি শুধু একটি নির্দিষ্ট আইডিকে ধরে এলিমেন্ট খুঁজে আনে।রিটার্ন করে: একটি নির্দিষ্ট এলিমেন্ট।

​getElementByClassName: এটি একই ক্লাসের যতগুলো এলিমেন্ট আছে সবগুলোকে একসাথে খুঁজে আনে। রিটার্ন করে: একটি HTMLCollection বা অ্যারের মতো লিস্ট।

​query selector : CSS-এ যেভাবে .class বা #id দিয়ে সিলেক্ট করা যায়, এখানেও সেভাবে এলিমেন্ট খোঁজা যায়। এটি শুধু প্রথম এলিমেন্টটি দেয়।

​query selector all : এটিও CSS সিলেক্টর ব্যবহার করে, তবে এটি ওই নামের যতগুলো এলিমেন্ট পায় সবগুলোকে একসাথে নিয়ে আসে। রিটার্ন করে: একটি NodeList।

3. What is Event Bubbling? And how does it work?

৩.​ইভেন্ট বাবলিং হলো জাভাস্ক্রিপ্টের একটি ন্যাচারাল বিহেভিয়ার। কোনো চাইল্ড এলিমেন্টে (যেমন বাটনে) ক্লিক করলে সেই ক্লিক ইভেন্টটি শুধু ওই বাটনেই থেমে থাকে না। এটি ধীরে ধীরে তার প্যারেন্ট, তারপর তার গ্র্যান্ড-প্যারেন্ট—এভাবে একদম উপরের রুট পর্যন্ত পৌঁছায়। একেই বলে বাবলিং।

4. What is Event Delegation in JavaScript? Why is it useful?

৪. ​ইভেন্ট ডেলিগেশন হলো সেই টেকনিক, যেখানে আমরা চাইল্ড এলিমেন্টে ইভেন্ট না বসিয়ে তাদের সবার কমন প্যারেন্ট এলিমেন্টে একটি মাত্র ইভেন্ট লিসেনার বসাই। বাবলিং-এর কারণে চাইল্ডে ক্লিক করলে প্যারেন্ট সেটা বুঝে ফেলে এবং আমরা event.target দিয়ে চেক করি কোন চাইল্ডে ক্লিক করা হয়েছে।

5. What is the difference between preventDefault() and stopPropagation() methods?

৫.

​preventDefault(): এটি কোনো এলিমেন্টের ডিফল্ট কাজ বন্ধ করে। যেমন: লিংকে ক্লিক করলে অন্য পেজে চলে যাওয়া বা ফর্ম সাবমিট করলে পেজ রিলোড হওয়া আটকানোর জন্য এটি ব্যবহার করা হয়ে থাকে।

stopPropagation(): এটি ইভেন্ট বাবলিং বন্ধ করে। অর্থাৎ, কোনো চাইল্ডে ক্লিক করলে সেই ইভেন্টটি আর উপরের প্যারেন্ট এলিমেন্টগুলোর কাছে না যাক, তখন এটি ব্যবহার করতে হয়।
